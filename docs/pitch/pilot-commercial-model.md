# Pilot Commercial Model & Strategic Licensing Structure

**Document Reference:** IDG-COM-MODEL-004  
**Scope:** Pilot and Staging Fleet Deployment Options  

---

## 1. Engagement Framework Options

We offer four licensing paths designed to accommodate varying procurement rules, security clearances, and operational models inside KRG and Federal branches:

| Model Option | Description | Financial Structure | Access Guidelines |
| :--- | :--- | :--- | :--- |
| **Option A: Pilot Advisory + Setup Fee** | Focuses strictly on initial system architecture, OIDC mapping workshops, and cloud dry-run setups. | Fixed setup fee; hourly consulting for integration mapping. | Standard staging instance access. |
| **Option B: SaaS License + Support** | Shared infrastructure managed on certified secure cloud environments with managed system updates. | Tiered monthly or annual licensing fee based on post volume. | Secure multi-tenant endpoints. |
| **Option C: Government-Hosted Deployment + Maintenance** | **[RECOMMENDED]** Full local partition deployment inside KRG government data centers under complete autonomous control. | Annual licensing and maintenance fee. Setup costs covered by government. | Fully dedicated sovereign nodes. |
| **Option D: Strategic Partnership / Revenue Share** | Hybrid deployment funded via small performance or clearance efficiency gains on transit volumes. | Revenue sharing structure based on transaction clearance volume. | Integrated public-private partnership. |

---

## 2. Recommended Pathway: Option C (Government-Hosted)
For optimal security compliance and absolute protection of KRG data assets, we recommend **Option C (On-Premises / Government-Hosted Deployment + Annual Support)**.

### Financial Partition Breakdown:
- **Infrastructure & Provisioning Costs:** Funded and managed directly by the contracting government department or integration partner (servers, database licenses, firewalls, and direct mTLS tunnels).
- **Core Software Licensing:** Covered by an annual Gateway Service Fee that guarantees access to system updates, patches, and schema changes.
- **Support, Training & SLA Service:** Fixed fee covering hands-on operator training, policy alignment compliance audits, and priority technical assistance.

---

## 3. Intellectual Property (IP) Boundaries
Unless explicitly negotiated and detailed in a bilateral treaty or formal contract:
- **All Platform Intellectual Property**—including core portal engines, sovereignty evaluation engines, multi-language libraries, and security auditing algorithms—remains the exclusive property of the platform owner.
- The government client is granted a non-exclusive, non-transferable, revocable license to execute and run the gateway software within designated regional partitions.
- Any custom connectors built to interface with external APIs (such as BRS or KRDPASS APIs) remain modular components of the platform library.
